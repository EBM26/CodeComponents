Variables

1. 'let' and 'cont' variables share in common the fact that they are blocked scoped, meaning they are only available inside a block of code defined by the curly brackets {}.

2. The main difference between 'let' and 'const' variables is that 'let' variables can be updated and their values change where as 'const' variables are just as they sound, constant, they can NEVER be updated.

3. RULE OF USE: This is a good practice to follow when writing varaiables in ES6:
A. Never use 'var'.
B. Use 'const' by default for all your variables.
C. Use 'let' only if you need to change the value of your variable later on in your code

Function Improvements: Arrows and Default Arguments

1. New "Fat Arrow Functions" are the first and only place we see a fat arrow in Javascript.

2. One-line arrow functions cannot have names (nameless functions). If you wish to be able to debug them, you should store them inside a variable like so:

const fullNames4 = names.map(name => `${name} bos`);

3. Due to the above rule, if you wish to use the 'this' keyword, you will HAVE to use a normal function in order that 'this' can reference the enclosed function and not the window object.

4. In normal Javascript, the 'this' keyword is change by the scope of each function. If there is a function within a function, 'this' will inherit it's scope and if it is nameless, it will return the window object.
In ES6, 'this' inherits the it's value from the parent function.

5. In ES6, you can pass default argument values in a function:

function calculateBill(total, tax = 0.13, tip = 0.15) {
return total + (total * tax) + (total * tip);
}

6. RULE OF USE: This is a good practice to follow when writing Arrow Functions in ES6:
A. Never use Arrow Functions if you want to use 'this'. Use regular functions instead.
B. Never use Arrow Functions if you need to use the 'arguments' object (a JavaScript object use when you need to use unlimited arguments in a function) .
C. Use 'Arrow Functions' on all your other simple and not-so-simple functions that do not need the above features to work.
